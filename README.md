# Intelligent System for Predicting Visitor Crowd Levels at Al-Masjid Al-Haram

A Streamlit dashboard that forecasts daily Umrah visitor crowd levels and recommends
optimal visiting times, powered by a **Growth-Rate XGBoost** model trained on Hijri-year
1445 data and tested on 1446.

This folder is a clean, deployment-ready version of the project extracted from the original
Google Colab notebook. It is ready for upload to GitHub and one-click deployment on
**Streamlit Community Cloud**.

---

## 2. How the app loads the model (no retraining at startup)

`app.py` loads the **pre-trained** model package from `models/growth_rate_xgboost_model.pkl`
through a cached loader (`get_model_package`, decorated with `@st.cache_resource`). This means:

- The model is loaded **once per running server**, not on every click/interaction.
- The model is **never retrained on startup** when the `.pkl` is present.
- As a safety fallback only, if the `.pkl` is missing (e.g. someone clones the repo without
  committing it), the app trains the model **exactly once** and saves it, then keeps using
  the cached copy. This is the only "absolutely necessary" case.

This is the only change made versus the notebook's `app.py` — a backend caching/self-heal
fix for deployment. **No UI, layout, colors, or text were changed.**

## 3. Run locally

```bash
# from inside the umrah-crowd-forecasting/ folder
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Step A — generate the model artifact ONCE (recommended)
python train_model.py
#   -> writes models/growth_rate_xgboost_model.pkl
#   -> writes outputs/growth_rate_xgboost_results_1446.xlsx

# Step B — launch the dashboard
streamlit run app.py
```

Open the URL Streamlit prints (usually http://localhost:8501).

## 4. Generate the trained artifacts (one command)

The model file is reproducible and deterministic (`random_state=42`). Generate it with:

```bash
python train_model.py
```

Expected console output ends with the final test metrics (MAE, RMSE, MAPE) for
**Growth-Rate XGBoost** on Hijri year 1446. After this runs you will have:

- `models/growth_rate_xgboost_model.pkl`
- `outputs/growth_rate_xgboost_results_1446.xlsx`

**Commit both files to GitHub** so Streamlit Cloud loads them directly and never needs to train.

## 6. Deploy on Streamlit Community Cloud

1. Push this folder to a **public GitHub repository** (e.g. `umrah-crowd-forecasting`).
   Make sure `models/growth_rate_xgboost_model.pkl` is committed (run `python train_model.py`
   first), otherwise the app will train once on its first boot.
2. Go to **https://share.streamlit.io** and sign in with GitHub.
3. Click **"Create app" → "Deploy a public app from GitHub"**.
4. Fill in:
   - **Repository:** `your-username/umrah-crowd-forecasting`
   - **Branch:** `main`
   - **Main file path:** `app.py`
5. (Optional) Under **Advanced settings**, set **Python version = 3.11** to match `runtime.txt`.
6. Click **Deploy**. Streamlit installs `requirements.txt` and launches `app.py`.

Your app will be live at:
`https://your-username-umrah-crowd-forecasting.streamlit.app`

To update later, just `git push` — Streamlit Cloud redeploys automatically.

## 7. Notes

- All file paths in the code are **relative**; there are no Colab or Google Drive
  dependencies at runtime.
- The dataset must keep its exact filename (the code locates it by name) and must stay at
  the repository root.
- If you retrain with different library versions than those used to create a committed
  `.pkl`, delete the old `.pkl` and re-run `train_model.py` so the saved model matches the
  installed `xgboost` version.
